1. Introduction & Core Concepts

info

A Note from the Developer​

Markd was built with aggressive error-handling and extensive safeguards — including the "Undo" time machine and strict physical sync checks — to ensure your bookmarks are always safe.

However, building a tool that perfectly bridges complex AI models, massive data trees, and live browser storage is a massive undertaking. As an independently developed application, it is possible that despite thorough testing, you may encounter a rare edge case or a minor bug.

Your data safety is the number one priority. If the app ever detects something it doesn't understand, it is designed to halt the operation and ask you to "Sync" rather than guess and make a mistake. If you do encounter unexpected behavior, please reach out with feedback! Your reports directly help improve the tool for everyone.

1. What is Markd?​

Markd is a professional, AI-powered bookmark manager built directly into your browser. Unlike traditional bookmark managers that trap your links in a proprietary database, Markd acts as a smart layer on top of your native Chrome bookmarks.

Everything you do in Markd — creating a folder, renaming a category, or moving a link — happens directly in your browser’s local storage. If you uninstall Markd tomorrow, your carefully organized bookmarks will remain perfectly intact in Chrome.

2. Browser Compatibility & Terminology​

Throughout this manual, the terms "Chrome" and "Browser" are used interchangeably.

  • Supported Browsers: Markd has been extensively tested and optimized for Google Chrome. However, because it is built on modern web standards, it is fully functional on almost all Chromium-based browsers (such as Microsoft Edge, Brave, Opera, Comet and Vivaldi). While we expect near 100% compatibility across these platforms, please note that minor browser-specific edge cases may exist.
  • Dynamic Root Folders: Different browsers use different names for their main bookmark areas (e.g., Chrome uses "Other Bookmarks", while Edge might use "Other Favorites"). Markd is smart enough to dynamically detect your specific browser's system folders. If you ever need to change where the app operates, go to Settings > Preferences > Root Folder Location, and the dropdown will automatically display the correct terminology for your specific browser.

3. The Strict 2-Level Hierarchy​

To keep your workspace clean and ensure the AI can sort with high accuracy, Markd enforces a strict 2-Level Hierarchy:

  1. Main Categories: Broad topics (e.g., Technology, Finance).
  2. Subcategories: Specific niches within a topic (e.g., Programming, Banking).

No Deep Nesting: The AI cannot reliably sort into folders that are 5 or 6 levels deep. If Markd detects deep folders already existing in your browser, it will generate "Flattening Proposals" to help you compress them into this clean 2-level system.

4. The Blueprint vs. Physical Reality​

To master Markd, you must understand the difference between the App Blueprint and Physical Reality.

  • Physical Reality (Chrome): These are the actual folders and bookmarks that currently exist in your Chrome Bookmark Manager.
  • The Blueprint (App Memory): This is Markd's "Architect's Plan." Markd allows you to design folders, import templates, and organize structures in the app before writing them to Chrome.

Because of this separation, Markd uses a color-coded status system so you always know if a folder exists in real life or just in the Blueprint.

5. The Visual Language (Color-Coded Statuses)​

When you look at the Category Tree, every folder has a status badge (e.g., [12 | 3], which means 12 total bookmarks and 3 subfolders). The color of the row tells you its exact state:

  • Active (Green text/badge) : A standard, physical folder that exists in Chrome and contains at least one bookmark.
  • Empty (Grey text/badge) : A physical folder that exists in Chrome, but currently contains 0 bookmarks. (Note: A Main folder is only considered "Empty" if it has 0 direct bookmarks AND all of its subfolders are also completely empty).
  • Plan (Blue text/badge) : A folder that exists only in Markd's Blueprint. It has not been physically created in Chrome yet.
  • Missing (Red text/badge, red left border) : A folder that Markd remembers in its Blueprint, but was deleted externally in Chrome.
  • Conflict / Proposal (Orange text/badge, orange left border) :
    • Flattening Proposal : AI-suggested folders to replace deep, nested hierarchies.
    • Naming Conflict : Markd detected multiple folders with the exact same name in Chrome. Crucial Note: Naming Conflicts are locked. You cannot rename a conflict in the app. You must either click the Trash icon to delete the conflict from the Blueprint and from the browser too, or click Apply Sync Plan to let Markd physically resolve the folders in Chrome. After syncing all options are unlocked for these folders.
  • Protected (Orange text/badge, orange border and background, icon) : A folder you have manually locked. Markd's AI completely ignores this folder. It cannot be deleted, sorted, or moved until you click the unlock icon.
  • Local Development (Cyan text/badge, cyan border and background) : A dedicated System Folder that safely catches localhost and private network URLs.
  • Broken Links (Red text/badge, red border and background) : A dedicated System Graveyard for dead URLs (404s, DNS failures).

6. UI Micro-Interactions​

To keep the interface clean, Markd hides action buttons until you need them.

  • Hover Reveal: Hover your mouse over any folder row to reveal the Action Menu on the far right. You will see icons to Add Subcategory (), Lock (), Sort Within (), and Delete ().
  • Smart Checkboxes: Checkboxes for bulk selection only appear when you hover over a row, or if the row is actively selected.
  • Disabled States: If an action is illegal (e.g., trying to lock a "Blueprint" folder that doesn't physically exist yet), the button will be completely removed from the row.

7. Smart Navigation & Visual Tracking​

When you perform an action (like moving bookmarks, renaming folders, or clicking Undo), Markd ensures you never lose your place in massive lists.

  • Auto-Scroll & Expand: After an operation finishes, the app automatically scrolls the workspace to the exact folder that was affected. If the item is hidden inside a closed folder, Markd automatically pops the drawer open to reveal it. If multiple items are affected across your workspace, the app intelligently scrolls to the top-most item on your screen.
  • The Green Highlight (Fade-on-Touch): To show you exactly what changed, affected folders flash with a bright Green highlight. This highlight is "smart": it will stay on the item indefinitely if you scroll away, but the moment you hover your mouse over the highlighted row (or after 4 seconds of looking at it), it smoothly fades out to keep the UI clean.
  • Parent Tracking Highlight: If you select a handful of subfolders or bookmarks, their parent Main Category row will gently glow with an Indigo highlight. This ensures that even if you close the drawer or scroll away, you can easily spot which Main Folders contain active selections.
Note on System Folders

System folders like Root, Local Development and Broken Links will still auto-scroll and expand to show you moved bookmarks, but they intentionally bypass the Green highlight to preserve their strict, color-coded warning styles. This exact same tracking behavior applies whenever you use the Undo / Redo buttons!

The Golden Rule

Always back up your bookmarks. Because Markd performs heavy, bulk physical operations on your Chrome bookmarks, you should regularly use the browser's native Bookmark Manager to export a full HTML backup. Markd also provides an Export Workspace Backup tool in the sidebar, but a native Chrome HTML backup is the ultimate failsafe. See further details and tips Part 9, Section 4: The Chrome Backup Anomaly.
