9. Sync, History & Maintenance

Markd operates on a "Dual-State" architecture: the App Blueprint and Chrome's physical reality. Because the app relies on Chrome's physical folders, the app tracks everything using unique internal IDs. If you make changes natively in Chrome while the app is closed, you must use the Sync Tools to update your view.

1. Sync Operations (The Core Loop)​

Because Markd is a full fledged Bookmark Manager it is highly advisable to use it instead of the browser's built-in manager. However, since you may occasionally edit a bookmark natively in the browser while Markd is closed, the app must constantly verify its Blueprint against reality.

  • Find & Refresh Plan (The Safe Update):
    • What it does: Scans your browser for new folders, updates bookmark counts, and checks for external changes.
    • Why it's safe: If you deleted a folder in Chrome, this button will not erase your hard work in the app. Instead, it turns the folder Red (Missing). This allows you to review what disappeared and either delete it from your Blueprint or restore it. If you renamed a folder in Chrome, Markd recognizes the ID and seamlessly updates the name in the app.
  • Clean & Resync Plan (The Strict Mirror):
    • What it does: Aggressively aligns the app with your browser's physical reality. It completely wipes out all "Missing" (Red) folders and unapplied "Plan" (Blue) folders from the app's memory. Use this when you want to discard your unsaved work and start fresh based exactly on what exists in Chrome.
  • Apply Sync Plan (The Exporter):
    • What it does: Pushes your Blueprint into Chrome. It physically creates your Blue folders and resolves Naming Conflicts.
    • The Unsaved Changes Indicator: Whenever you modify your Blueprint (e.g., creating a Blue "Plan" folder or triggering an Orange "Conflict"), the Sync Operations header and the Apply Sync Plan button at the bottom of the section will instantly turn Orange and the button itself begin to gently pulse. This glowing beacon is your reminder that you have unsaved structural changes in the app that need to be pushed to Chrome.
    • Dialog Options: When clicked, a dialog asks how you want to apply the plan:
      1. Create Only Folders with Bookmarks: The app will only physically create your planned folders if they actually contain bookmarks. This keeps your Chrome bookmark bar clean and free of empty folders.
        Note: Physically existing empty folders will also be removed.
      2. Create Complete App Structure: Forces Chrome to build every single folder in your Blueprint, even if they are 100% empty or orphans.
  • Auto-Sync Cooldown (Edge Cases):
    • If you have Auto-Sync enabled, the app watches Chrome for changes in real-time and syncs them instantly in the background without requiring manual button clicks. However, to prevent the app from freezing, it uses a slight delay (cooldown) to group rapid changes together. If you are doing massive, lightning-fast reorganizations natively in Chrome, the auto-sync might miss a beat.
      Best Practice: Make it a habit to click Find & Refresh Plan regularly to guarantee your screen matches reality.

2. The Undo / Redo Time Machine​

Mistakes happen. Markd features a robust, state-aware Command Manager capable of rolling back massive operations.

  • How it Works: Every time you move a bookmark, rename a folder, or run a bulk AI sort, Markd takes a snapshot of the exact DOM state and folder IDs.
  • Surgical vs. Full Refresh:
    • For small actions (moving a few bookmarks), clicking Undo will "surgically" update the UI in real-time, pulling the bookmark back and animating the changes seamlessly.
    • For massive actions (like an AI Sort of 100+ items), clicking Undo will trigger a Progress Bar, physically revert the bookmarks in Chrome, and then perform a full, clean UI refresh to ensure 100% accuracy.
  • Healing Orphans: If you delete a newly created folder by clicking Undo, Markd is smart enough to revert that folder back to a red "Missing" or blue "Plan" state in the Blueprint, rather than just breaking the interface.

3. Import & Export​

Markd offers two entirely different types of backup systems to protect your data.

  • Export Category Tree (The Blueprint):
    • What it does: Downloads a .json file containing only your folder names and structure (The Blueprint). It contains zero bookmarks.
    • Why use it? To share your perfect folder setup with a friend, or to back up your structure before running a "Delete All Categories" reset.
  • Export Workspace Backup (The Time Capsule):
    • What it does: Downloads a massive .json file containing your structure and every single bookmark inside them.
    • Safety Note: Protected folders () are strictly ignored. Markd refuses to read or back up anything inside a locked folder.
    • Restore Workspace Backup: (Premium Feature). Clicking this will give you a final, terrifying warning. If you proceed, Markd will wipe your active Chrome root (preserving only folders) and rebuild it perfectly from the backup file.

4. The Chrome Backup Anomaly (Important)​

Google Chrome allows you to create multiple folders with the exact same name at the same level. However, when you use Chrome's native Bookmark Manager to export an HTML backup, Chrome automatically merges folders with the same name together.

Markd prevents this. If Markd detects two folders with the same name, it generates an orange Naming Conflict and keeps them safely separated.

tip

Best Practice for Backups: To capture your workspace exactly as it appears on your screen—even if you have unresolved orange Naming Conflicts—use Markd's Export Workspace Backup tool. This creates a precise "Time Capsule" of your current setup, keeping those identically named folders safely separated just as you see them.

Important Limitations of Markd's Backup:

  • It only backs up the Active Root you are currently working in (e.g., Other Bookmarks). It does not back up your entire browser.
  • It strictly excludes Protected Folders (). Markd's exporter refuses to read or save anything inside a locked folder.

When to use Chrome's Native HTML Backup instead: If you need a 100% complete backup of your entire browser across all roots (Bookmarks Bar, Mobile, etc.), or if you need to ensure your Protected Folders are safely stored, you should use Chrome's built-in Bookmark Manager export tool. Just keep in mind that if you use the Chrome exporter while you still have orange Naming Conflicts, Chrome will merge those identically named folders together in the final HTML file.

tip

Save the complete bookmark tree with all the bookmarks using Markd only:

  1. In "Settings", under "Root Folder Location", select the first root folder.
  2. Run "Celan & Resync Plan".
  3. Click "Export Workspace Backup" and then choose "Download Markd Backup".
  4. Switch to the next root folder in Settings and repeat the steps for each root folder.
note

If you have protected folders, temporarily remove their protection before creating the backup so those folders are included. Once the backup is complete, re-apply the protection to the folders.

5. Resolving "Out of Sync" Errors​

If you rename or delete a folder using Chrome's native Bookmark Manager while Markd is closed, and then try to interact with that same folder inside Markd, a dialog will appear: "Folder Exists in the browser."

Because Markd's Blueprint doesn't match Chrome's physical reality anymore, it halts your operation to prevent data corruption. Simply click Sync & Cancel Operation in the dialog. Markd will instantly update its UI to match Chrome, and you can safely retry your action.

6. The Danger Zone​

Located at the bottom of the sidebar, these tools are for deep cleaning.

  • Delete Empty Folders: Scans your entire tree and silently deletes every physical folder that contains 0 bookmarks. (It will skip System and Protected folders).
  • Delete All Categories: Wipes the Blueprint in the app, but leaves your physical folders in Chrome untouched.
  • Delete All Folders & Categories: A full nuke. It will prompt you asking what to do with the bookmarks (Move to Root or Delete), and then destroy the entire managed structure.
