4. Managing Folders

1. Creating Categories​

  • Main Categories: Click the large + Add Main Category button in the sidebar. The input field is displayed on the top of the category tree. If valid folder name is entered, the new folder will appear in the category tree sorted alphabetically, but it is always strictly below system folders like Local Development or Broken Links.
  • Subcategories: Hover over an existing Main Category and click the icon on the right.
  • If you type a name that already exists at the actual level, the input box will turn red, block the save, and show an error. Press Escape to cancel creation or continue typing to create a unique name.

2. Renaming Folders​

To rename a folder, simply click its text and start typing. Press Enter or click away to save.

  • Duplicate Prevention: You cannot rename a folder to match an existing folder at the same level.
  • Naming Conflicts are Locked: If Markd detects a "Naming Conflict" (two folders with the identical name in Chrome), you cannot rename it inside the app. The app automatically uses an internal renaming method to add a number [1] to the end of duplicate folder names. You can either click the icon to delete these folders from the Blueprint and browser, or click Apply Sync Plan in the sidebar to force Chrome to create them using these unique names."

3. Flattening Explained (Deep Folders)​

  • Flattening Deep Folders (Proposals): Chrome allows infinite folder nesting, but Markd enforces a clean 2-level system. If the app detects deep folders (e.g., Recipes > Desserts > Cakes), it creates orange Flattening Proposals to help you compress them.
    • Single Flattening (Clicking the icon): If you click the flatten icon on a proposal row, Markd automatically grabs that folder and all of the deeper folders beneath it. It moves all the bookmarks up into a clean 2-level structure and safely deletes the old, messy branch in one smooth action.
    • Bulk Flattening (Using Checkboxes): For massive deep folder trees, you can use the checkboxes to hand-pick exactly which proposals to keep. By default, selecting an item also selects all items nested beneath it, effectively selecting the entire branch. Once you've made your selection, the Floating Action Bar offers two options:
      • Accept: Flattens only the items you checked. Important: If you leave checkboxes unchecked within that same branch, a warning dialog will appear telling you exactly how many bookmarks and folders are about to be permanently deleted. This ensures you never accidentally lose data by missing a checkbox.
      • Reject: Deletes the selected proposals and completely wipes the underlying physical folders and bookmarks from your browser. A detailed warning will calculate exactly what is about to be destroyed before you confirm.

4. Drag & Drop Mechanics​

Markd features a robust drag-and-drop engine designed for heavy organizational work, supporting both single items and massive bulk operations:

  • Nesting (Demotion): Drag a Main Category into another Main Category to instantly demote it into a Subcategory.
  • Promoting: Drag a Subcategory into the empty workspace area (a blue "Drop here to promote" banner will appear). Dropping it will instantly promote it to a top-level Main Category.
  • Bulk Drag & Drop: Once you have multiple folders checked, simply click and drag any of the selected rows. The Floating Action Bar will temporarily hide, and the app will bundle your selection into a single visual badge (e.g., "☰ 6 Folders"). You can then drop the entire batch to promote or demote them all at once!
  • Edge Scrolling: When dragging an item, move your cursor to the extreme top or bottom edge of the list. The screen will automatically smooth-scroll, allowing you to move items across massive lists.
  • Spring-Loaded Folders: While dragging a bookmark or folder, hover over a closed folder for 1.2 seconds. It will automatically pop open, allowing you to drop the item deep inside.
  • Illegal Moves: The cursor will turn into a restricted "not allowed" symbol (and bulk drags will be safely cancelled) if you attempt an illegal move, such as:
    • Dragging a Main Category that contains subfolders into another Main Category.
    • Dragging a subfolder into its own parent.
    • Dragging an item into a Protected folder.
    • Dragging a folder into the Broken Links graveyard (bookmarks are allowed, but folders are blocked).
    • Dragging Orphans, Proposals, or Conflicts.

5. Resolving Drag & Drop Conflicts​

If you drag a folder into a destination that already has a folder with the exact same name, Markd will pause the drop and present a Conflict Resolution Dialog with three options:

  1. Merge: Moves all bookmarks and subfolders from your dragged folder into the existing one, then deletes the original empty shell.
  2. Rename & Move / Rename & Promote: Automatically appends a [1] (or next available number) to your dragged folder's name and completes the move.
  3. Overwrite Orphan: If the destination folder is a blue/red "Orphan" (meaning it doesn't physically exist in Chrome), this option allows you to replace that ghost definition with your real, physical folder.
Bulk Conflicts

If you are dragging a batch of folders and multiple naming conflicts occur, a checkbox will appear at the bottom of the dialog allowing you to "Apply this choice to all remaining conflicts" in that batch, saving you from clicking through multiple warnings!

6. Bulk Selection & The Floating Action Bar (FAB)​

When you need to reorganize massive amounts of folders, doing it one by one is tedious. Markd provides a powerful bulk selection engine tailored specifically for folder management.

The Global Selection Controls​

At the very top right of your workspace (above the category tree), you will find a row of Global Selection Checkboxes. Because different types of folders require different actions, Markd enforces Selection Modes. You can only select one "type" of folder at a time:

  • Main Folders: Selects only top-level categories.
  • Subfolders: Selects only secondary folders.
  • Empty Folders: Selects any folder in your tree that contains zero bookmarks.
  • Main Orphans & Suborphans: Selects Blue/Red folders that only exist in the app's Blueprint.
  • Flat Proposals & Naming Conflicts: Selects problem folders that need your attention.
note

If you try to manually check a box for a Subfolder while a Main Folder is already selected, the app will block the click and remind you to clear your current selection first.

The Floating Action Bar (FAB)​

Once you select one or more folders, the Floating Action Bar (FAB) smoothly slides up from the bottom of the screen. The buttons on this bar dynamically change based on exactly what type of folders you selected:

  • Contextual Selection (The Magic Button): If you manually check a subfolder (e.g., "Taxes" inside "Finance"), the FAB will instantly generate a smart button saying "Select all in 'Finance'". Clicking this will automatically check every other subfolder inside that specific parent. Clicking it again deselects them.
  • Rename... : Opens the advanced Bulk Rename tool. Instead of typing new names manually, this tool allows you to:
    • Find & Replace: Swap specific words across dozens of folders at once.
    • Add Prefix/Suffix: Instantly add tags like [Archive] to the beginning or - 2024 to the end of all selected folders.
  • Sort Within... : (Appears only for Main Categories). Allows you to trigger the AI to sort the bookmarks inside all of the selected folders simultaneously.
  • Accept / Reject: (Appears only for Proposals). Allows you to bulk-flatten deep folders, or safely reject them from your plan entirely.
  • Delete: Deletes the selected batch. Markd is incredibly safe here: if it detects that any of the folders you are about to delete contain bookmarks, the danger dialog will intercept the action and ask if you want to rescue those bookmarks by moving them to the Root or Parent folder before the deletion happens.
  • Clear: Instantly deselects all folders, clears the checkboxes, and hides the FAB. Pressing the Escape key on your keyboard does this as well.

6. Making Blueprints & Missing Folders Real​

As you know, Blue (Plan) and Red (Missing) folders are "Orphans" — they exist only in the app's memory, not physically in the browser. However, Markd is designed to seamlessly materialize them into real folders the moment you interact with them. Once created, they will instantly follow the standard color-coded language (turning Green if they have bookmarks, or Grey if they are empty).

  • Renaming an Orphan: If you rename a Blue or Red folder, Markd instantly assumes you want to keep it. It will physically create the folder in Chrome under the new name, and the folder will immediately turn Grey (Empty).
  • Adding Subfolders: If you hover over a Main Orphan category and click Add Subcategory ( ), the app will physically create the new subfolder in Chrome. Because a subfolder cannot exist without a parent, Markd will automatically create the Main Orphan in Chrome as well!
  • Dropping Items Inside: If you drag and drop a bookmark or a folder into an Orphan, Markd will instantly build that folder structure in the browser so it can safely catch and store your dropped items.
  • The Sub-Orphan Chain Reaction: If you interact with a Subcategory that is an Orphan, and its Main Category parent is also an Orphan, Markd will safely build the entire branch (Parent -> Child) behind the scenes to save you time.
tip

Rule of Thumb: Basically, any action you perform on an Orphan folder — except clicking the Delete icon ( ) — will automatically bring it to life in your browser.
